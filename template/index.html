HTML

Mission: to simplify the markup language and to depart from XHTML

  • <!doctype html>, <meta charset=utf-8>
  • No need to use quotes on attribute values
  • New semantic elements and ARIA accessibility attributes
  • Data attributes: <span data-icon=nice>
  • No need to close pair elements: <b>, <br>...
  • <a> can cover block element(s)
  • No need to specify type of the <script> and <link>
  • Descriptive link relations
  • New form features!

New HTML Forms

  • Form fields can have types
  • New <input> types
    search, tel, url, email, datetime, date, month, week, datetime-local, number, range, color
  • Completely new form elements
    <datalist>, <keygen>, <output>, <progress>, <meter>
  • Note: on a mobile device the type can constrain virtual keyboard setting
  • See http://wufoo.com/html5

Fallbacks, Polyfills
How to deal with the legacy?


Polyfill is downloadable code which provides facilities that are not built-in to a web browser.

For example, many features of HTML5 are not supported by versions of Internet Explorer older than version 8, but can be used by web pages if those pages install a polyfill. Polyfills can also be used to add entirely new functionality to browsers.

Html5Forms polyfill: html5Forms.js

Apache Cordova

It enables software programmers to build applications for mobile devices using JavaScript, HTML5 and CSS3, instead of lower-level languages such as Objective-C, Java or C#

Features

  • Accelerometer
  • Camera
  • Compass
  • Contacts
  • File
  • Geolocation
  • Media
  • Network
  • Notification: Alert, Sound, Vibration
  • Storage
